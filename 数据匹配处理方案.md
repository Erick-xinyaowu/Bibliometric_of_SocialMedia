# 数据处理方案：基于aweme_id匹配并计算时间差

## 目标
基于 `aweme_id` 字段匹配两个CSV文件，向 `douyin_data_with_topics.csv` 中新增两个字段。

## 文件说明

### 输入文件

**文件1: `douyin_data_with_topics.csv`**
- 文件路径: `data/douyin_data_with_topics.csv`
- 记录数: 评论数据（多条评论对应一个视频）
- 关键字段: `aweme_id`（视频ID）, `create_time`（评论发布时间）
- 其他字段: `comment_id`, `content`, `user_id`, `like_count`, `所属地区`, `bertopic_topic_id`, `topic_label`

**文件2: `search_contents_2026-04-15_formatted.csv`**
- 文件路径: `data/search_contents_2026-04-15_formatted.csv`
- 记录数: 视频搜索数据（45条）
- 关键字段: `aweme_id`（视频ID）, `create_time`（视频发布时间）
- 其他字段: `title`, `desc`, `user_id`, `nickname`, `liked_count`, `collected_count`, `comment_count`, `share_count`

### 输出文件
- 文件路径: `data/douyin_data_with_topics_enriched.csv`
- 将在 `douyin_data_with_topics.csv` 基础上新增两个字段

## 处理逻辑

### 步骤1: 读取两个CSV文件
```python
df_comments = pd.read_csv('data/douyin_data_with_topics.csv')
df_videos = pd.read_csv('data/search_contents_2026-04-15_formatted.csv')
```

### 步骤2: 基于aweme_id进行左连接
```python
# 使用左连接，保留所有评论数据，即使视频ID在search_contents中不存在
df_merged = df_comments.merge(
    df_videos[['aweme_id', 'create_time']],
    on='aweme_id',
    how='left',
    suffixes=('', '_video')  # 视频的create_time重命名为create_time_video
)
```

### 步骤3: 新增字段1 - base_time
- **字段名**: `base_time`
- **数据来源**: `search_contents_2026-04-15_formatted.csv` 中的 `create_time` 字段（视频发布时间）
- **数据类型**: 日期格式 (YYYY-MM-DD)
- **说明**: 该字段记录视频的原始发布时间

### 步骤4: 新增字段2 - delta_T
- **字段名**: `delta_T`
- **计算公式**: `douyin_data_with_topics.create_time - search_contents_2026-04-15_formatted.create_time`
- **数据类型**: 整数（天数）
- **说明**:
  - 正值: 表示评论在视频发布后X天才出现
  - 负值: 理论上不应该存在（评论时间早于视频发布时间）
  - 空值: 当 `aweme_id` 在 `search_contents_2026-04-15_formatted.csv` 中找不到匹配记录时

## 数据匹配情况预判

- **完全匹配**: 当评论的 `aweme_id` 在 `search_contents_2026-04-15_formatted.csv` 中存在时，`base_time` 和 `delta_T` 将有有效值
- **部分匹配**: 当评论的 `aweme_id` 在 `search_contents_2026-04-15_formatted.csv` 中不存在时，`base_time` 和 `delta_T` 将为空值（NaN）

## 注意事项

1. **一对多关系**: 一个视频ID可能对应多条评论记录，所有该视频的评论将共享相同的 `base_time`
2. **空值处理**: 如果 `aweme_id` 在 `search_contents_2026-04-15_formatted.csv` 中找不到匹配，`base_time` 和 `delta_T` 将为空值
3. **日期计算**: `delta_T` 将以天数为单位计算（如: 8 表示评论在视频发布8天后出现）
4. **编码格式**: 输出文件将使用 `utf-8-sig` 编码，确保中文显示正常
